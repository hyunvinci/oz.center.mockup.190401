명훈 책임님[2019.02.26]


개인적인

설비 모니터링 시나리오를 단순 모니터링에서

SmartDn, Things 리소스, 사용자와 연계하는 서비스로 가정하여 보았습니다


업무에 참조하여 주세요


1.

서버측 관리자 웹 인터페이스에서 SmartDn장치를 등록 할 수 있다 (가상의 등록 / 실 장비가 없어도 등록 가능)

등록의 결과로 SmartDn은 고유의 ID를 갖게 된다 (고유ID 에는 다음과 같은 구조가 연결된다 / SmartDn 별칭, 연결된 사용자, 연결된 Thing 리소스)

고유한 SmartDn ID는 서버측에 위치한 MQTT Broker의 인증설정으로 즉시 업데이트된다 이를 통해 각 SmartDn의 MQTT Pub/Sub을 제한한다


2.

서버측 관리자 웹 인터페이스에서 Thing 리소스를 등록 할 수 있다 (실제 발생데이터 접근용 / MQTT Topic과 읽거나 쓸 메시지 Key 값등)

(이 리소스 정보는 SmartDn에 설치된 TAS에 사전 등록된 Thing 리소스와 동일하게 입력한다)


3.

서버측 관리자 웹 인터페이스에서 등록된 Thing 리소스를 SmartDn 장치로 할당한다 (이는 해당 SmartDn의 Things 리소스 접근 권한 정의)


4.

서버측 관리자 웹 인터페이스에서 SmartDn을 특정 사용자에게 할당한다 (해당 사용자로 웹 인터페이스 로그인해야 Dashboard를 볼 수 있음)


5.

SmartDn과 Thing 리소스가 정의 및 할당되면 해당 Thing 리소스로부터 데이터를 Sub하여 DB에 저장한다 (기존 프로메테우스 역할)


6.

서버측 관리자 웹 인터페이스에서 데시보드 항목에서 데시보드 위젯을 생성한다

위젯은 위 Thing 리소스 또는 DB 기반으로 실시간 데이터 또는 과거 데이터를 획득하여 챠트를 표시한다


7.

서버측 사용자 웹 인터페이스로 사용자가 로그인하고 모니터링을 진행
